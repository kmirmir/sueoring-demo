### 3. 큐 기반 순차 처리 시스템 구현 ⭐
**가장 중요한 개선 사항**

#### 문제점
- 빠르게 여러 제스처를 하면 중간 제스처들이 누락됨
- TTS 재생 중 새 제스처를 하면 충돌 발생
- 자막은 쌓이지만 음성은 순차 재생 안됨

#### 해결 방법: Queue 기반 아키텍처

```typescript
// 1. 제스처 큐 관리
const gestureQueueRef = useRef<Array<{text: string, timestamp: number}>>([]);
const isProcessingQueueRef = useRef(false);

// 2. 긴급 제스처 우선 처리
const addGestureToQueue = (gesture: string) => {
  const isEmergency = gesture.includes('위험') ||
                      gesture.includes('119') ||
                      gesture.includes('도와주세요');

  if (isEmergency) {
    gestureQueueRef.current.unshift(gestureItem);  // 큐 맨 앞
  } else {
    gestureQueueRef.current.push(gestureItem);     // 큐 맨 뒤
  }

  processQueue();
};

// 3. 순차 처리 (Promise 기반)
const processQueue = async () => {
  if (isProcessingQueueRef.current) return;

  isProcessingQueueRef.current = true;
  const gestureItem = gestureQueueRef.current.shift();

  setSubtitleHistory(prev => [...prev, gestureItem]);
  await speakTextAsync(gestureItem.text);  // TTS 완료까지 대기

  isProcessingQueueRef.current = false;
  processQueue();  // 다음 제스처 처리
};
```

#### 주요 기능
1. **모든 제스처 인식** - 누락 없이 큐에 저장
2. **완벽한 동기화** - 자막과 TTS가 항상 일치
3. **긴급 우선 처리** - 위험/119 등은 큐 맨 앞으로
4. **실시간 피드백** - 현재 처리 중 제스처 표시 (▶ 안녕하세요)
5. **큐 상태 표시** - 대기 중인 제스처 개수 (우측 상단 노란 배지)
6. **사용자 제어** - 마지막 취소, 전체 초기화 버튼

#### UI 추가
- **큐 오버레이** (우측 상단): "대기 중: 3개"
- **현재 처리** (하단): "▶ 안녕하세요"
- **큐 컨트롤 버튼**:
  - 🔵 "↩ 마지막 취소" - 큐의 마지막 제스처 삭제
  - 🔴 "🗑️ 전체 초기화" - 큐 전체 삭제 + TTS 중지

### 4. 브라우저 자동 열기 문제 해결 🌐

#### 문제
- `start http://localhost:8081` 명령이 간헐적으로 작동 안 함
- 포트 충돌 시 대체 방법 필요

#### 해결책 (4가지 방법 제공)

##### 방법 1: HTML 자동 감지 페이지 (추천) ⭐
**파일**: `quick-open.html`
- 더블클릭만으로 실행
- 4개 포트 자동 감지 (8081, 8082, 19006, 19007)
- 실행 중인 서버를 찾아 자동으로 브라우저 열기
- 시각적으로 예쁜 UI

```html
<!-- JavaScript로 포트 자동 감지 -->
async function checkPort(port) {
  const response = await fetch(`http://localhost:${port}`, {
    mode: 'no-cors'
  });
  return true;
}
```

##### 방법 2: Node.js 헬퍼 스크립트
**파일**: `scripts/open-browser.js`
- 서버가 준비될 때까지 자동 대기 (최대 30초)
- 크로스 플랫폼 지원 (Windows, Mac, Linux)

```bash
npm run web:open
```

##### 방법 3: Windows 배치 파일
**파일**: `open-web.bat`
- 더블클릭으로 실행
- 4개 포트 순차 시도

```batch
start http://localhost:8081
start http://localhost:8082
start http://localhost:19006
start http://localhost:19007
```

##### 방법 4: PowerShell 스크립트
**파일**: `open-web.ps1`
- 포트가 열려있는지 확인 후 열기
- 더 스마트한 감지

```powershell
.\open-web.ps1
```

#### NPM Scripts 추가
**파일**: `package.json`

```json
{
  "scripts": {
    "web:open": "node scripts/open-browser.js",
    "web:dev": "npm run web & npm run web:open"
  }
}
```

**사용법**:
```bash
# 서버 시작 + 브라우저 자동 열기
npm run web:dev

# 이미 실행 중인 서버에 브라우저만 열기
npm run web:open
```
